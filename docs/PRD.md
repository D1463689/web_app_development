# 產品需求文件 (PRD)：讀書筆記本系統

## 1. 專案概述
- **背景與動機**：學生在學習過程中經常需要閱讀大量課外讀物或參考用書，但常常讀過就忘，缺乏一個系統化的工具來記錄閱讀軌跡、整理思緒以及回顧所學。本專案旨在打造一個專屬學生的輕量級讀書筆記系統。
- **目標用戶**：學生（包含國高中生至大學生）。
- **核心價值主張**：提供簡單、直覺的介面，幫助學生輕鬆記錄每本書的閱讀歷程與心得，建立個人知識庫，提升閱讀效益。

## 2. 功能需求
以下列出本系統的主要功能（共 5 項），並以使用者故事（User Story）格式描述：

1. **書籍資訊記錄**
   - 作為 **學生**，我希望 **能夠新增書籍的基本資訊（如書名、作者、分類）**，以便 **管理我正在閱讀或計畫閱讀的書單**。
2. **撰寫讀書心得**
   - 作為 **學生**，我希望 **能夠在每本書籍下撰寫並儲存心得筆記**，以便 **記錄書中的重點與我的啟發**。
3. **書籍評分系統**
   - 作為 **學生**，我希望 **能夠為讀完的書籍給予評分（例如 1 到 5 顆星）**，以便 **未來可以快速篩選出推薦的好書或回顧價值**。
4. **閱讀狀態與進度追蹤**
   - 作為 **學生**，我希望 **能夠標記書籍的閱讀狀態（如未讀、閱讀中、已完讀）**，以便 **清楚掌握目前的閱讀進度與目標**。
5. **書單搜尋與篩選**
   - 作為 **學生**，我希望 **能夠透過關鍵字（書名）或分類來搜尋書籍**，以便 **在累積大量書籍紀錄後能快速找到特定資訊**。

## 3. 非功能需求
- **技術限制**：本專案為保持輕量化與開發速度，後端將採用 `Flask` 框架，前端介面使用 `Jinja2` 模板引擎進行渲染，資料庫則使用免伺服器設定的 `SQLite`。
- **效能考量**：操作應保持順暢，頁面載入速度快，資料儲存與讀取的延遲感應極低。
- **安全考量**：需防範常見的安全弱點（如 SQL Injection 隱碼攻擊及 XSS 跨網站指令碼攻擊），確保使用者填寫的筆記與資料能安全地儲存與顯示。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增書籍與基本資訊（書名）。
  - 記錄與顯示對應書籍的讀書心得。
  - 針對書籍進行評分（1-5 顆星）。
- **Should Have (應該有)**：
  - 書籍與心得的編輯及刪除功能（CRUD 完整操作）。
  - 書籍的閱讀狀態標記。
- **Nice to Have (可以有)**：
  - 關鍵字搜尋與分類篩選。
  - 閱讀進度追蹤或讀書數據統計圖表。
  - 多圖片或連結附件上傳功能。
