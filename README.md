## 系統簡介
本專題旨在開發一套輕量級的 Web 應用系統，透過 Python Flask 框架建構後端服務，並結合 Jinja2 模板引擎與 SQLite 資料庫，實現前後端整合的完整應用。

系統核心功能包含資料的新增（Create）、查詢（Read）、更新（Update）與刪除（Delete），使用者可透過網頁介面即時操作資料，無需直接接觸資料庫，大幅提升使用便利性與操作安全性。

在系統設計上，採用 MVC（Model-View-Controller）概念進行模組化開發：

Model：負責資料庫結構與資料存取（SQLite）
View：透過 Jinja2 動態渲染 HTML 頁面
Controller：使用 Flask 處理路由與應用邏輯

本系統主要目標使用者為初學 Web 開發之學生或需進行簡易資料管理的使用者，透過簡潔的操作流程與清楚的功能設計，達到學習與實務應用兼具的目的。

此外，本專題亦強調開發流程的完整性，從需求分析、系統設計到實作與測試，皆依序進行，培養系統開發的基本能力。
## 技術棧
前端：
Jinja2 模板引擎（負責動態產生 HTML，提升畫面與資料的整合效率）

後端：
Python + Flask（負責處理 HTTP 請求、路由設計與應用邏輯）

資料庫：
SQLite（輕量級關聯式資料庫，適合小型應用開發與測試）

## 個人心得
在本次專題開發過程中，我對 Web 應用程式的整體架構有了更深入的理解，特別是在前後端整合與資料流設計方面，收穫相當豐富。透過 Flask 框架的實作，我學會如何規劃路由、處理使用者請求，並將資料傳遞至前端進行顯示。

此外，在使用 SQLite 的過程中，也熟悉了基本的資料庫操作，包括資料表設計、SQL 查詢語法以及資料的 CRUD 操作，使我對資料持久化有更具體的認識。

在開發過程中，遇到的主要困難包括：

前後端資料傳遞不一致，導致頁面顯示錯誤
路由設計初期混亂，影響系統可讀性與維護性

針對這些問題，我透過重新整理程式架構、參考官方文件以及逐步除錯，成功改善系統穩定性與可讀性。
